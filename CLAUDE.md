# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Bootstrap (cold-start / new machine)

```zsh
bash bootstrap/bootstrap.sh             # apply everything (idempotent)
bash bootstrap/bootstrap.sh --dry-run   # preview without writing
```

Prerequisites: `jq`, `git`, `bash`. Restart Claude Code after running.

## Adding a skill

```zsh
mkdir skills/<name>
# write skills/<name>/SKILL.md with frontmatter + content (see format below)
bash bootstrap/bootstrap.sh   # picks up the new symlink
```

## Skill format

```markdown
---
name: skill-name
description: One-line trigger description — shown in skill list, used for invocation matching.
---

# Content here
```

Skills can also be packaged as `.skill` files — ZIP archives containing `SKILL.md` plus an optional `evals/` directory. **The bootstrap does not auto-symlink `.skill` packages** (it only globs directories); extract or install these manually.

## Architecture

### Symlink model

Everything is symlinked — nothing is copied:
- `skills/<name>/` → `~/.claude/skills/<name>` (install-skills.sh)
- `hooks/*.sh` → `~/.claude/scripts/*.sh` (install-hooks.sh)
- `config/cc-statusline.sh` → `~/.config/iterm2/cc-statusline.sh` (install-skills.sh)

The bootstrap sub-scripts (`bootstrap/lib/`) handle their domains independently; `bootstrap.sh` just sequences them.

### Hooks

Three hooks, each versioned in comments:

**`on-prompt.sh`** (UserPromptSubmit) — two tiers:
- Tier 1 (always): scoring classifier emits task type + token budget hint (`plan/implement/refactor/debug/review/read/general`)
- Tier 2 (conditional, skipped for `low` budget): memory injection (once per session per project, via `/tmp/claude-ctx/` flag files) + git branch/dirty state for high-stakes tasks

**`on-session-end.sh`** (Stop) — session stats + memory write-back nudge + git uncommitted warning. Emits a `systemMessage` JSON so Claude Code displays the summary. Reads `~/.claude/scripts/mem-map.conf` for project→memory routing.

**`post-tool-use.sh`** (PostToolUse, matcher: Edit|Write) — lightweight grep-only antipattern scanner: innerHTML with non-literal RHS, Linux `/home/` paths on macOS, SQL string interpolation. Emits `systemMessage` JSON on findings only.

### mem-map.conf

**Machine-specific and not version-controlled.** Generated by `install-hooks.sh` on first run at `~/.claude/scripts/mem-map.conf`. Maps project `basename` → memory filename. Edit this file to add project-specific memory routing; the template is embedded in `install-hooks.sh`.

### statusline (cc-statusline.sh)

Single `jq` pass extracts `cwd`, `context_window.used_tokens`, `context_window.max_tokens`. Maps CWD → project key via `REL="${CWD#$HOME/Code/}"` case match. Project → skill hint abbreviations live in the `HINTS` associative array inside the script — update there when adding projects.

`install-settings.sh` writes the PostToolUse hook entry into `~/.claude/settings.json` (surgically via `jq`, never overwrites existing config, backs up before mutating).
